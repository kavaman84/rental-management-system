# 群晖NAS部署出租屋租赁管理系统

## 📦 快速部署（推荐）

### 步骤1: 准备工作

确保群晖DSM已安装：
- Container Manager（容器管理器）
- Docker Compose

### 步骤2: 上传文件

使用File Station创建文件夹 `/docker/rental-system`，然后上传：
- `Dockerfile`
- `docker-compose.yml`
- `package.json`
- `server-sqlite.js`
- `public/` 文件夹及其所有内容

### 步骤3: 启动服务

#### 方法A: 使用图形界面（最简单）
1. 打开Container Manager
2. 点击"容器" → "创建"
3. 选择"从docker-compose文件创建"
4. 选择 `docker-compose.yml` 文件
5. 点击"创建"

#### 方法B: 使用SSH命令行
```bash
cd /docker/rental-system
docker-compose up -d
```

### 步骤4: 访问系统

打开浏览器访问:
```
http://192.168.1.5:4000
```

登录凭据:
- 用户名: `admin`
- 密码: `admin123`

## 📝 详细说明

### 1. 端口说明

- 默认端口: `4000`
- 修改端口: 编辑 `docker-compose.yml`，修改 `ports` 配置
- 例如: `"8080:4000"` 则访问 `http://192.168.1.5:8080`

### 2. 数据持久化

数据库文件存储在: `/docker/rental-system/data/rental_system.db`

### 3. 常用命令

```bash
# 查看容器状态
docker-compose ps

# 查看日志
docker-compose logs -f

# 重启服务
docker-compose restart

# 停止服务
docker-compose down

# 启动服务
docker-compose up -d
```

## 🎯 功能说明

### 收据管理

## ⚙️ 群晖特殊配置

### 1. 防火墙设置

确保群晖防火墙开放以下端口：
- HTTP端口（默认4000）

### 2. 网络设置

- 默认使用bridge网络模式
- 可以访问局域网内其他设备

### 3. 资源限制

在 `docker-compose.yml` 中添加资源限制：

```yaml
deploy:
  resources:
    limits:
      cpus: '1.0'
      memory: 512M
    reservations:
      cpus: '0.5'
      memory: 256M
```

## 🔧 故障排查

### 问题1: 容器无法启动

```bash
# 查看日志
docker-compose logs

# 常见原因：
# - 端口被占用
# - 文件权限问题
# - 依赖未正确安装
```

### 问题2: 无法访问系统

```bash
# 1. 检查容器是否运行
docker ps

# 2. 检查端口映射
docker-compose ps

# 3. 检查群晖防火墙

# 4. 尝试重启容器
docker-compose restart
```

### 问题3: 数据库文件丢失

确保使用了volume挂载：
```yaml
volumes:
  - ./data:/app/data
```
